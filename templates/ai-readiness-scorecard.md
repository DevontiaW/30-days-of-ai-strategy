# AI Readiness Scorecard

*Score your organization across five dimensions to understand where you're strong, where you're exposed, and what to invest in before launching AI initiatives.*

---

## Instructions

For each dimension, read the level descriptions and honestly select the one that best describes your organization **today** — not where you plan to be or where you were last quarter. Circle or highlight your level.

---

## 1. Data

| Level | Description |
|-------|-------------|
| **1 - Ad hoc** | Data lives in spreadsheets, email, and individual systems. No single source of truth. Reports require manual assembly from multiple people. |
| **2 - Siloed** | Core systems (CRM, ERP, etc.) exist but aren't integrated. Each department has its own data. Cross-functional analysis requires heroic effort. |
| **3 - Integrated** | Key data sources are connected. A central data warehouse or lake exists. Standard definitions for core metrics. Some data quality monitoring. |
| **4 - Managed** | Data is cataloged, governed, and has clear ownership. Quality is measured and managed. Self-serve analytics available. Data is treated as an asset. |
| **5 - Optimized** | Data is a managed product with SLAs. Real-time pipelines. Advanced governance. Data democratization across the org. Proactive quality management. |

## 2. Talent

| Level | Description |
|-------|-------------|
| **1 - No capability** | No data science or ML talent. Analytics is basic reporting done by business analysts or finance. |
| **2 - Emerging** | 1-2 data-savvy people who do analysis, maybe some Python. No dedicated ML roles. Talent is spread thin across projects. |
| **3 - Building** | Small data science team (3-5 people). Can build and deploy basic models. Some ML in production. Dependent on specific individuals. |
| **4 - Established** | Dedicated ML/AI team with clear roles (data engineers, ML engineers, data scientists). MLOps practices in place. Can hire and retain talent. |
| **5 - Advanced** | Full AI/ML organization with specialized roles. Research capability. Strong employer brand for AI talent. AI literacy across business teams. |

## 3. Infrastructure

| Level | Description |
|-------|-------------|
| **1 - Manual** | No cloud infrastructure for ML. Models run on someone's laptop. No version control for data or models. |
| **2 - Basic** | Some cloud services. Manual deployment processes. No ML-specific infrastructure. Ad hoc compute for training. |
| **3 - Developing** | Cloud ML platform (SageMaker, Vertex, Azure ML) in use. Basic CI/CD for models. Some monitoring. Manual but documented processes. |
| **4 - Mature** | Full MLOps pipeline. Automated training, testing, deployment. Model monitoring and drift detection. Feature store or equivalent. |
| **5 - Advanced** | Sophisticated ML platform. A/B testing infrastructure. Real-time serving. Automated retraining. Cost optimization. Multi-model management. |

## 4. Culture

| Level | Description |
|-------|-------------|
| **1 - Resistant** | Leadership skeptical of AI. "We've always done it this way." Data-driven decisions are rare. Fear of automation replacing jobs. |
| **2 - Curious** | Interest from leadership but no action. Some individuals experimenting. No organizational support or budget for AI. |
| **3 - Supportive** | Leadership sponsors AI initiatives. Budget allocated. Willingness to experiment. But AI is still "the tech team's thing." |
| **4 - Data-driven** | Decisions across the org are informed by data and analytics. Cross-functional collaboration on AI projects. Experimentation is encouraged and supported. |
| **5 - AI-native** | AI thinking is embedded in how the org operates. Business teams propose AI use cases. Continuous learning culture. Failure is treated as learning. |

## 5. Governance

| Level | Description |
|-------|-------------|
| **1 - None** | No AI-specific policies. No inventory of AI systems. No review process. Nobody owns AI risk. |
| **2 - Reactive** | Policies created in response to incidents. Some awareness of AI risk. Ad hoc reviews. No consistent standards. |
| **3 - Basic** | AI inventory exists. Risk classification for use cases. Basic documentation standards. Review process for high-risk projects. |
| **4 - Structured** | Formal governance framework. Model cards for all production systems. Regular audits. Bias testing. Clear escalation paths. Incident response plan. |
| **5 - Embedded** | Governance is part of the development lifecycle, not a gate. Automated compliance checks. Proactive risk management. External audits. Regulatory readiness. |
