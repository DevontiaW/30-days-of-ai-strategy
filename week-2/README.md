# Week 2: Implementation

The goal this week is execution. By Friday, you should have a concrete pilot plan — scoped, staffed, measured, and with stakeholder buy-in.

| Day | Topic | Key Takeaway |
|-----|-------|-------------|
| [Day 08](day-08.md) | Designing an AI Pilot That Actually Proves Value | Pilots need a hypothesis, a baseline, and a decision gate — not just a demo |
| [Day 09](day-09.md) | Building the Right Team | Start with 3-5 people and a clear problem, not a full data science department |
| [Day 10](day-10.md) | Technical Architecture Decisions | Design your pilot architecture for throwaway, not permanence |
| [Day 11](day-11.md) | Stakeholder Management & Expectation Setting | Map influence and support — then engage each quadrant differently |
| [Day 12](day-12.md) | Success Metrics That Matter | If your metrics dashboard doesn't include a dollar figure, you're tracking science projects |
| [Day 13](day-13.md) | The Valley of Disillusionment | Every AI initiative hits a trough — the ones that survive have someone translating "we're learning" for the board |
| [Day 14](day-14.md) | Week 2 Reflection + Pilot Readiness Checklist | Consolidate your plan with a comprehensive readiness assessment |
